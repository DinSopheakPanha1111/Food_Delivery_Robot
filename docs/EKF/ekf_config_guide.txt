odomN_differential : When set to true, instead of using the robot's absolute position
it looks at how much position has changed over time (like speed). This helps avoid errors
when the odometry data is not perfectly accurate.

imuN_differential : When set to true, the robot looks how much IMU measurements (like angular speed)
have changed over time instead of the absolute values

odomN_relative : When set to true, it only resets the measurements relative to the first sensor reading

imuN_relative : When set to true, it only resets the measurements relative to the first sensor reading
